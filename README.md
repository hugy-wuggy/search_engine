	Search Engine — Поисковый движок для локальных документов

	Описание проекта

Search Engine — это консольное приложение на C++ для полнотекстового поиска по локальным текстовым документам. Программа строит инвертированный индекс, обрабатывает поисковые запросы, рассчитывает релевантность и формирует ранжированный список результатов. Проект реализован в рамках финальной работы курса «C++ Developer с нуля» от Skillbox.

	Основной функционал

- Индексация текстовых документов (в многопоточном режиме)
- Поиск по запросам с ранжированием результатов
- Расчёт абсолютной и относительной релевантности
- Автоматическое формирование файла с результатами поиска (answers.json)
- Модульное тестирование (Google Test)

	Стек технологий

- C++ 20
- CMake
- Google Test — модульное тестирование
- nlohmann/json — работа с JSON
- Многопоточность — индексация выполняется в потоках

	Архитектура проекта

- ConverterJSON — читает конфигурацию и запросы из JSON-файлов, записывает результаты
- InvertedIndex — строит инвертированный индекс из документов
- SearchServer — выполняет поиск, рассчитывает релевантность и возвращает результаты
- tests/ — модульные тесты, реализованные с помощью Google Test

	Инструкция по запуску

	Системные требования

- C++ компилятор (MinGw)
- CMake ≥ 3.31
- Git
- Поддержка C++20

	Установка и запуск

`bash
# 1. Клонируйте репозиторий
git clone https://github.com/ваш-логин/search_engine.git
cd search_engine

# 2. Создайте директорию для сборки
mkdir build && cd build

# 3. Соберите проект с помощью CMake
cmake ..
cmake --build .

# 4. Запустите приложение
./search_engine

Запуск модульных тестов

# Внутри директории build
ctest

Формат входных файлов

config.json — список документов:


{
  "files": [
    "resources/file001.txt",
    "resources/file002.txt"
  ]
}

requests.json — список поисковых запросов:


{
  "requests": [
    "example query",
    "another example"
  ]
}

answers.json — автоматически создаваемый файл с результатами поиска.
